# Source registry — regulators + law portals per domain + jurisdiction

> **Starting-point registry, NOT a citation source.** Rows below are where research begins. Never cite this file — or a bare homepage — as authority in a spec.

How to use:

1. Find your domain table, then your jurisdiction row.
2. Open the entry-point URL; navigate / search to the specific provision.
3. Extract facts per `01-regulatory-research-workflow.md` Phase 3+.
4. Capture a full citation block (specific URL + article + version + retrieved date) per `03-citation-conventions.md`. Durable per-statute citations live there (source-registry pattern), not here.

Deliberate constraints (v2.0.0):

- **Entry-point URLs only** — regulator homepages, official law-portal roots, gazette roots. Deep links (statute pages, PDFs, sub-pages) were removed: they rot fastest. Pinned deep URLs belong in citation blocks, where each carries its own retrieved date.
- **Last verified** is honest, not aspirational. `2026-05` carries the v1 blanket claim "URLs verified 2026-05" — rows were NOT individually re-verified for v2. `2026-06 (added, unverified)` marks rows added in this rewrite. `—` means authority recorded, no URL on file. Re-verify a row when you run R4 for that jurisdiction; update the date.
- If a URL breaks: search "<jurisdiction> <authority name>", land on the official page, fix the row.
